# Осенний парадокс: почему заказов стало больше, а средний чек упал?

Проект посвящён анализу данных о бронировании билетов на онлайн-сервисе за период с 01.06.2024 по 31.10.2024.

**Дата:** 03.06.2026

**Статус:** завершён.

**Стек:** Python (Pandas, NumPy, Matplotlib, SciPy (t-тест, U-критерий, тест Левена), исследовательский анализ данных (EDA) с проверкой гипотез, Jupyter Notebook.

**Данные:** обезличенные данные о заказах (что, когда, сколько, на каком устройстве купили), события (что за мероприятие, где и кто организатор) и курсы валют.

**Цель:**

- выяснить, почему осенью увеличилось количество заказов, но снизилась их средняя стоимость;
- определить лидеров среди событий и площадок;
- сопоставить активность пользователей мобильных устройств и стационарных компьютеров.

**Задачи:**

- загрузить и предобработать данные; 
- провести исследовательский и статистический анализ; 
- сформулировать выводы и дать рекомендации.

**Ключевые этапы и решения:**

| Этап|Альтернативное решение|Мое решение|Пояснение |
|---|---|---|---|
| Обработка пропусков в категориальных столбцах | Удаление строк с пропусками | Замена пропусков в категориях на `unknown` | Удаление строк снижает объём данных и может внести смещение, особенно если пропуски неслучайны. `unknown` сохраняет наблюдения и явно маркирует неопределённость — это корректно для анализа (в т. ч. при расчёте метрик и группировках в Pandas/SQL). |
| Расчёт интервала между покупками (аналог `days_since_prev`) | Использование «сырого» столбца без учёта пользователя или порядка событий | Пересчёт интервала через сдвиг даты внутри пользователя (например, с помощью `groupby` + `shift` в Pandas или оконных функций `LAG`/`OVER` в SQL) | «Сырой» столбец не учитывает порядок событий по пользователю и может давать некорректные интервалы. Сдвиг внутри пользователя даёт точный расчёт времени между соседними покупками для каждого клиента, что критично для анализа повторных покупок и когорт. |
| Подготовка данных по выручке для агрегации | Анализ выручки в разных валютах «как есть» без приведения к единой базе | Конвертация всей выручки в рубли для единообразия анализа | Сравнение и суммирование выручки в разных валютах без конвертации искажает результаты (особенно при агрегациях, MAU/выручка, A/B‑тестах). Приведение к одной валюте позволяет корректно считать общие метрики, средние и проводить статистические тесты. |
| Очистка данных от выбросов | Жёсткое удаление явных ошибок/выбросов по фиксированным порогам | Фильтрация по 99‑му процентилю | Жёсткие пороги могут отсечь легитимные, хоть и редкие, кейсы (крупные заказы, премиум‑пользователи), что смещает средние показатели. 99‑й процентиль мягко ограничивает «крайние» значения, сохраняя основную массу данных и снижая влияние экстремальных выбросов на метрики и тесты. |
| Сегментация и анализ поведения пользователей | Объединение всех данных и расчёт общих метрик без разбивки | Разделение анализа по валютам и устройствам на ранних этапах | Если сразу агрегировать всё вместе, важные различия между сегментами «растворяются» (например, мобильные пользователи могут вести себя иначе, а выручка в USD может иметь другую динамику). Ранняя сегментация помогает увидеть паттерны, корректно спланировать A/B‑тесты и выбрать релевантные метрики для каждой группы. |
| Проверка гипотез о различиях между группами (например, мобильные vs десктоп) | Использование только t‑критерия без проверки предпосылок | Использование непараметрического теста (Манна‑Уитни) наряду с t‑критерием | t‑критерий требует нормальности распределения и (в классической версии) равенства дисперсий; при нарушении предпосылок его p‑value может быть некорректным. Манн‑Уитни не зависит от нормальности и хорошо работает для порядковых данных и распределений с «тяжёлыми хвостами» (типично для интервалов между заказами, выручки и т. п.), давая более надёжную проверку гипотез. |

**Выводы:**

- Осенью растёт спрос из‑за старта спортивных лиг, театрального сезона и предновогоднего периода.
В дни акций пользователи делают больше заказов за визит, а на массовые категории (концерты, ёлки, театр) снижают цены — это тянет средний чек вниз.
Также есть недельная цикличность: в середине недели растёт объём покупок, но падает средний чек, а в выходные — наоборот.
- По доле заказов лидируют концерты (в любой сезон), осенью драйверами продаж становятся спортивные мероприятия, ёлки и театр.
Основной объём продаж даёт Каменевский регион и партнёр «Билеты без проблем».
- Пользователи чаще покупают с мобильных устройств и летом, и осенью.
A/B‑тесты указывают, что мобильные пользователи делают больше заказов, но в эксперименте есть проблемы (пересечение групп, дисбаланс), поэтому выводы требуют осторожности.

**Рекомендации:**

- Запускать акции в дни пиков спроса (начало/конец месяцев).
- Осенью снижать цены на массовые категории, сохранять маржу в нишах типа стендапа.
- Усилить работу с Каменевским регионом и партнёром «Билеты без проблем», поддерживать лояльность в высокомаржинальных регионах.
- Предлагать менее маржинальные товары в середине недели, дорогие — в выходные.
- Пересмотреть дизайн A/B‑теста и увеличить выборку в группе B.

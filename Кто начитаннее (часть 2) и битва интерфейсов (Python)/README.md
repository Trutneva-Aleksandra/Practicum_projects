# Кто начитаннее и битва интерфейсов.

Тетрадь содержит 2 проекта, показывающие навыки проверки гипотез в Python и составления аналитической записки.

## ПРОЕКТ 1. Кто начитаннее: москвичи или петербуржцы?

**Дата:** 18.05.2026

**Статус:** завершён.

**Стек:** Python (pandas, matplotlib, scipy, statsmodels).

**Задача:** проверить гипотезу, что пользователи из Санкт‑Петербурга проводят в среднем больше времени за чтением и прослушиванием книг в приложении, чем пользователи из Москвы.

### Контекст и гипотеза

Жители Санкт‑Петербурга более вовлечены в чтение/прослушивание книг, поэтому средняя активность (в часах) в приложении у них выше.

- **H₀:** μСПб ≤ μМосква (средняя активность в СПб не больше, чем в Москве).
- **H₁:** μСПб > μМосква (активность в СПб статистически значимо больше).

### Ключевые этапы и решения

1. **Предобработка данных.** Обнаружены пользователи, присутствующие в обеих группах (возможная ошибка разметки). Для соблюдения независимости выборок такие записи удалены (5.56 % данных).
2. **Разведочный анализ (EDA).** Построена гистограмма распределения часов активности. Выявлена выраженная правосторонняя асимметрия и выбросы — это повлияло на выбор статистического теста.
3. **Выбор метода.** Из‑за ненормальности распределения обоснован выбор непараметрического теста Манна‑Уитни (альтернатива — t‑тест с оговорками). В коде реализованы оба подхода для сравнения.
4. **Статистическая проверка.** Проведён односторонний тест с уровнем значимости α = 0.05.
5. **Интерпретация и выводы.** На основе p‑value сформулирован вывод относительно H₀ и предложены возможные причины отсутствия значимых различий.

### Результаты

- Размер выборок: Москва — 5990 пользователей, Санкт‑Петербург — 2306.
- Медианная активность: Москва — 0.888 ч, СПб — 0.875 ч (разница минимальна).
- p‑value t‑теста ≈ 0.326 (> 0.05): нет оснований отвергать нулевую гипотезу.
- Вывод: статистически значимых различий в средней активности между городами не выявлено.

## ПРОЕКТ 2. Битва интерфейсов: A/B‑тест нового дизайна сайта

**Статус:** завершён

**Автор:** Александра Трутнева

**Стек:** Python (pandas, matplotlib, statsmodels, scipy)

**Задача:** оценить влияние упрощённого интерфейса на конверсию в покупку в течение 7 дней после регистрации.

### Контекст и гипотеза

Интерфейс интернет‑магазина был признан сложным по отзывам пользователей. Разработана новая версия сайта. Цель A/B‑теста — проверить, приведёт ли новый дизайн к росту конверсии в покупку.

- **H₀:** p_A = p_B (конверсии в группах A и B значимо не различаются).
- **H₁:** p_A < p_B (конверсия в тестовой группе B выше, чем в контрольной A).

Целевой эффект: рост конверсии минимум на 3 %. Уровень значимости α = 0.05, мощность 80 %.

### Ключевые этапы и решения

1. **Проверка корректности теста.** Исключены пользователи, участвующие в нескольких тестах одновременно (12.21 % строк). Распределение между группами A/B приведено к допустимому балансу (49.7 % / 50.3 %).
2. **Построение событийного пайплайна.** Объединены таблицы участников и событий, найдена дата регистрации, рассчитан лайфтайм, оставлены события за первые 7 дней.
3. **Расчёт метрик.** Конверсия считалась на уровне пользователя (`nunique(user_id)`), чтобы избежать перекоса из‑за многократных сессий.
4. **Оценка мощности и размера выборки.** Через `statsmodels` рассчитан минимально необходимый размер выборки (3761 на группу) и подтверждено, что данные удовлетворяют этому требованию.
5. **Статистический тест.** Проведён Z‑тест для пропорций с проверкой предпосылок ($np > 10$).
6. **Интерпретация.** Сопоставлены статистическая значимость и практический эффект относительно целевого показателя.

### Результаты

- Контрольная группа (A): 4952 пользователя, 1377 покупок → конверсия 27.81 %.
- Тестовая группа (B): 5011 пользователей, 1480 покупок → конверсия 29.54 %.
- Рост конверсии: +1.73 п.п. (относительный рост покупателей ≈ 7.48 %).
- p‑value Z‑теста ≈ 0.028 (< 0.05): различия статистически значимы.
- Целевой рост (≥ 3 %) не достигнут, хотя статистическая значимость есть.

### Выводы и рекомендации

Статистически значимый рост конверсии подтверждает позитивное влияние нового интерфейса. Однако практический эффект ниже целевого уровня. Рекомендуется:
- провести дополнительный анализ когорт/сегментов (устройства, источники трафика),
- оценить долгосрочные метрики (повторные покупки, LTV),
- при необходимости доработать интерфейс и повторить тест с уточнённой гипотезой.
